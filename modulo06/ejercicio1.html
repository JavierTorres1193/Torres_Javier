A- Describe dos usos o funciones que proporciona la declaración DOCTYPE en una página web.

Informa al navegador qué versión de HTML (o XML) se usó para escribir el documento. Doctype es una declaración no una etiqueta. Además, podemos referirnos a ella como "document type declaration" o por las siglas "DTD".

B-Para que se utilizan las siguientes etiquetas:

"Head" = El elemento HTML "head" provee información general (metadatos) acerca del documento, incluyendo su título y enlaces a scripts y hojas de estilos. Ninguna. Si el elemento es un srcdoc de un "iframe" , o si la información de título está disponible desde un protocolo de nivel superior, cero o más elementos de metadatos

"Title" = El elemento "title" HTML define el título del documento que se muestra en un browser la barra de título o la pestaña de una página. Solo contiene texto; las etiquetas dentro del elemento se ignoran. Contenido de metadatos

"Meta" = Sirve para aportar información sobre el documento.. Sus etiquetas son: "meta" (solo tiene una). Está definido como: Elemento de cabecera. Crea una caja

"Style" = Es el elemento encargado de indicar la información de estilo. Está definido como: Elemento de cabecera.

"Body" = La etiqueta HTLM "body" se utiliza para definir el cuerpo del documento HTML. Específicamente, es la parte en donde se describe el contenido de la página, colores, forma, estructura, texto, imágenes.

"Encabezados" = Los encabezados HTML o HTML Header Tags son las etiquetas que conocemos como H1, H2, H3, H4, H5 y H6. Estas etiquetas de título se emplean dentro de un sitio web para identificar los temas y subtemas de los que trata la página.

"a" = Es una etiqueta que nos ayuda a poder crear un enlace a una página web. El atributo principal de la etiqueta HTML es href, donde pondremos el enlace al que queremos conectar. Otro atributo muy usado es target, el cual nos sirve para indicar si el enlace se abrirá en una nueva ventana o en la misma.

"Strong"> = Definición. El elemento strong es el apropiado para marcar con especial énfasis las partes más importantes de un texto.

"br" = El elemento HTML line break "br" produce un salto de línea en el texto (retorno de carro). Es útil para escribir un poema o una dirección, donde la división de las líneas es significante. No utilices "br" para incrementar el espacio entre líneas de texto; para ello utiliza la propiedad margin de CSS o el elemento "p"

"ul" = El elemento ul define una lista desordenada. El elemento contiene uno o más elementos li que especifican los ítems de la lista. Por el contrario, el elemento ol define una lista ordenada. El elemento contiene uno o más elementos li que especifican los ítems de la lista como una secuencia ordenada.

"li" = Las etiquetas OL y LI nos sirven para crear listas, OL para listas ordenadas y UL para listas sin orden. Dentro de las listas, los elementos se identifican con la etiqueta LI.

"p"= sirven para crear los párrafos y dentro de ellas también pueden indicarse atributos específicos como la alineación del texto, ya sea izquierda, derecha, centrada o justificada.

"span" = Sirve para aplicar estilo al texto o agrupar elementos en línea. Está definido como: Elemento especial, y por lo tanto en línea. Crea una caja: En línea.

"table" = Las tablas más sencillas de HTML se definen con tres etiquetas: "table" para crear la tabla, "tr" para crear cada fila y "td" para crear cada columna. La etiqueta "table" encierra todas las filas y columnas de la tabla.

"Tr" = El elemento HTML tr inserta una fila en una tabla (elementos HTML table). En las tablas HTML, las filas son los contenedores para las celdas y no las columnas. Por lo tanto, este elemento debería contener un número de celdas que muestren información (elemento HTML td) o encabezados (elemento HTML th).

"Td" = El elemento HTML Celda de tabla ("td") define la celda de una tabla que contiene datos. Esta participa en el modelo de tablas .

"img" = Este elemento incluye atributos globales. Alineamiento de la texto respecto al contexto que la rodea. Este atributo define el texto alternativo que describe la imagen, texto que los usuarios verán si la URL de la imagen es errónea o la imagen tiene un formato no soportado o si la imagen aún no se ha descargado.

c- La etiqueta "script" puede ir dentro de las etiquetas de de "body" o "head", hay alguna diferencia donde pueda ir?

-El script debe ubicarse en el body para optimizar los tiempos de carga de la pagina, hay en casos particulares en donde es conveniente ponerlos en el header con async o defer, de este modo, los scripts son obtenidos inmediatamente cuando empieza el parseo pero sin bloquear el mismo.
